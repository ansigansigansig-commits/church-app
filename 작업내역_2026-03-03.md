## 1차 수정 — 총 인원 집계 버그

### 증상
출석 입력 화면의 총/남/여 인원 집계가 항상 0으로 표시됨

### 원인
Firebase Realtime Database가 순차적 숫자 키(1~89)를 가진 객체를 **JSON 배열로 변환**하는 동작 때문.
배열의 빈 인덱스에 `null`이 채워지고, `updateSummary()`에서 `null.present` 접근 시 TypeError 발생하여 집계가 0으로 남음.

### 수정 내용 (3곳)
1. **`normalizeFirebaseData()` 함수 추가** — Firebase 폴링 시 배열을 객체로 변환하고 null 항목 제거
2. **`updateSummary()`** — `if (v.present)` → `if (v && v.present)` null 가드
3. **`updateStats()`** — `if (!val) return;` null 가드 추가

## 3차 수정 — 통계 로직 버그

### 증상
체크 안 한(미기록) 교인은 통계에 안 잡히고, 결석 사유를 입력한 교인만 통계에 나옴

### 원인
`updateStats()`에서 출석/결석 기록이 있는 사람만 `total++` 처리하여, 아무 체크도 안 된 교인은 `total=0`으로 필터링됨

### 수정 내용
- **이전**: 개별 기록 기준 — 기록 있는 사람만 `total++`
- **이후**: 전체 세션 기준 — 출석 기록이 있는 세션 수를 모든 교인에게 동일하게 `total`로 적용
- 미체크 교인도 해당 세션에서 미출석으로 정확히 반영됨
- 예: 세션 4개, A교인 2번 출석 체크 → `2/4회 (50%)`
